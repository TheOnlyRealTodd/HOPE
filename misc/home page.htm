Higher Order Programming Environment

What is HOPE?

HOPE is an application development system.  Similar to LEGO's, users (both programmers and non-programmers) build tailored applications from smaller, self-connecting components (called receptors) by "snapping" them onto a graphical virtual surface--no coding is required.  Receptors perform the functions of:

  • acquiring data from external sources (such as a stock ticker web service)
  • processing the data
  • "visualizing" the raw or processed data (text, graphically, speech, notifications, alerts, etc.)
  • data storage / retrieval

A HOPE application is built by selecting the desired receptors and configuring them (again no coding is required) based on the user's data source needs, processing, storage, and visualizations.

Why should I use HOPE?

The most common use-case for HOPE is real-time analysis and visualizations for data streams such as stock prices, social media posts, transactions, traffic, RSS feeds, weather, and so forth.

Key Features:

A key feature of HOPE is its adaptability.  HOPE enables users to:

  1. Incorporate new data sources into existing analysis applications
  2. Plug in new visualizations and apply them immediately to existing data sources
  3. Distribute analysis workloads seamlessly and securely across networks
